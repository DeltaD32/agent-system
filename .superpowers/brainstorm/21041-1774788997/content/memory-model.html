Agent Memory & Vault Structure

Section 2 of 4 — AgentVault at ~/Documents/AgentVault

Vault Folder Layout

📂 AgentVault/
📂 shared/
← all agents read + write
   project notes, findings,
   decisions, glossary

📂 instructions/
📄 project-manager.md
📄 coder.md
📄 researcher.md
📄 sysadmin.md
📄 graphic-designer.md
📄 … (one per role)
← role system prompts & rules
   agents read-only their own file

📂 agents/
📂 project-manager/
📂 coder/
📂 researcher/
📂 sysadmin/
📂 … (one per role)
← private per-agent memory
   read/write own folder only

📂 system/
← orchestrator metadata
   agent registry, task log
   agents read-only

Permission Matrix

Agent shared/ instructions/
own		 agents/
own		 agents/
others		 system/
PM RW R RW R RW
Archivist RW R RW R RW
Coder RW R RW — R
All others RW R RW — R
RW = read + write  ·  R = read-only  ·  — = no access
PM + Archivist can read all agent folders to coordinate cross-agent work

What goes in each agent's private folder

agents/coder/
📄 context.md — current project context
📄 snippets.md — reusable code patterns
📄 toolchain.md — preferred libs & tools
📄 session-log.md — append-only task history

How agents check memory before acting

1. Read instructions/{role}.md → load role rules & constraints
2. Read agents/{role}/context.md → load private memory
3. Search shared/ → check if answer already exists
4. Execute task → write findings back to shared/ or own folder
5. Archivist periodically indexes & deduplicates shared/ knowledge
