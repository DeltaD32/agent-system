Key design decisions

  • One asyncio task per hired agent — no separate processes, no message broker daemon, agents are lightweight coroutines
  • Redis pub/sub — orchestrator pushes events (agent moved, task started, message sent) to frontend over WebSocket in real time
  • Obsidian vault is the memory layer — agents read/write markdown files; no vector DB needed (Obsidian search handles retrieval)
  • LLM routing by task complexity — simple tasks stay local (Ollama), complex reasoning routes to Claude API
  • SearXNG auto-discovery — checks localhost:8080, then your Docker network; spawns a container if neither is reachable
